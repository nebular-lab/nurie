# nurie お散歩アプリ 要件

## 概要

歩いた経路を自動で記録する iPhone アプリ。バックグラウンドで常時記録し、過去に歩いたすべての経路を地図上で振り返れる。

## 制約

- iPhone のみ対応
- **無料の範囲内**で開発・利用 (Free Apple ID、有料 Apple Developer Program は使わない)

## 機能要件

### F-1. 経路記録 (バックグラウンド)

- アプリを開いていない / 画面ロック中でも、歩いた経路を自動で保存する
- スタート / ストップ操作は不要 (常時バックグラウンドで動作)
- 位置情報は端末ローカルに保存する
- 位置取得設定:
  - 精度: `Accuracy.Balanced`
  - 距離間隔: 10m (`distanceInterval: 10`)

### F-2. 過去経路の表示

- アプリ起動時に地図を表示する
- 過去に歩いたすべての経路を地図上に重ねて 1 枚で表示する (フィルタなし)
- **色分け**:
  - 過去の経路: 古いほど薄く、新しいほど濃いグレースケールグラデーション
  - 今日の経路: 別色 (例: 赤) で目立たせる
- データ保持期間: **無制限** (削除機能は今回は実装しない)

### F-3. 画面構成

- **1 画面で完結する** (タブ・モーダル・別ページなし)
- 地図がメインビュー
- 補助 UI:
  - 現在地ボタン (右下、再センター用)
- 補助 UI として実装しないもの:
  - 速度・距離・時間表示 (不要)
  - 日付フィルタ・経路一覧
  - 設定画面

### F-4. 地図の初期表示

- 起動時の中心: 現在地
  - 位置取得前 / 拒否時のフォールバック: 最後に記録した点 → なければ全経路フィット → なければ日本全体
- ズームレベル: 16 (街区レベル)
- 移動中の自動追従はしない (ユーザーが地図をパンしている時に勝手に動かない)

### F-5. 権限リクエスト UX

- 初回起動時、いきなり OS の「常に許可」ダイアログを表示する
- 拒否された場合は地図のみ表示し、経路記録は機能しない (再起動時に再度要求)

## 非機能要件

- データはローカル保存 (サーバーなし)
- オフライン動作 (モバイル通信なしでも記録・閲覧できる)
- バッテリー消費を考慮した位置取得頻度 (F-1 の設定値で担保)

## スコープ外 (今回は実装しない)

- 歩数計測・HealthKit 同期 (有料プログラム必須のため)
- 月間目標歩数・進捗表示 (歩数機能を持たないため)
- iOS ホーム画面ウィジェット (有料プログラム必須のため)
- 速度・距離・時間の表示
- 経路の削除・編集
- 経路の日付フィルタ・一覧表示
- ユーザー認証・クラウド同期
- 経路の共有・SNS 連携

## 既知の運用上の制約

### 7 日サイクル問題

- Free Apple ID で署名したアプリは **7 日で証明書が切れる**
- 切れたら Mac で再ビルドし、USB 経由で iPhone に再インストールする必要がある
- 散歩に出かけた時に切れていると起動できない可能性がある
- 許容運用とする (週 1 回程度の再インストールを前提)

### データロスのリスク

- 7 日サイクルの再インストールで「上書き」できれば SQLite データは保持される
- ただし署名コンフリクト等で「アプリ削除 → 再インストール」が必要になった場合、保存していた経路はすべて失われる
- バックアップ機能は今回は実装しないため、**経路データは消える可能性がある**ことを許容する

### バックグラウンド動作の信頼性

- iOS は省電力のためバックグラウンドタスクを停止することがある
- 完全な連続トラッキングは保証されない (途切れる前提で設計する)
- 取得できなかった区間は地図上で「途切れた線」として表示される

## 想定するライブラリ

- `expo-location` - 位置取得 (foreground / background 両対応)
- `expo-task-manager` - バックグラウンドタスク登録
- `expo-sqlite` - ローカル保存 (位置点列を効率的に扱える)
- `react-native-maps` - 地図表示 (iOS は Apple Maps)
- `expo-router` - ルーティング (1 画面なので最小利用)
